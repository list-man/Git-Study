git branch -a 显示所有远程分支和本地分支
git branch -d [branche1][branch2][branch3] 删除分支（必须依据merge过)
git branch -D 删除分支（忽略merge状态)

//push之后总是origin---远程服务器代码仓库
git branch <branchname>	创建分支
git branch <branchname> 切换到分支
git push -u <remote> <branchname> 将分支提交到origin,并建立与的关联.这里只能是origin，不能是子分支
git push origin --delete <branchname> 从远程服务器将分支删除
git branch -r -d <branchname> 从远程服务器将分支删除


创建分支，可选项表示可以给予本地哪一分支来创建分支，默认是当前所在的分支！
git branch <newbranchName> [-t | --track | --set-upstream | --no-track] [refBranchName]


git merge srcBranch [destBranch]如果没有destBranch，那么就合并到当前分支

git status 显示当前分支的状态


如果你觉得你合并后的状态是一团乱麻，想把当前的修改都放弃，你可以用下面的命令回到合并之前的状态：
git reset --hard HEAD

git status   查看索引内容


//详细见git config -h帮助
git config --global user.name "JayGuo"	创建或修改全局配置文件节点
git config --global --unset user.id	删除节点
